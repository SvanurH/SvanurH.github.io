Git学习

创建仓库

  • git init 直接新建一个仓库
  • git clone 从远程服务器上克隆一个已经存在的仓库

Git的工作区域和文件状态

flowchart LR
工作区 -->|git add| 暂存区 -->|git commit| 本地仓库
  • 工作区：资源管理器中的文件夹
  • 暂存区：临时存储区域，用于保存即将提交git仓库的修改内容
  • 本地仓库： 通过git init命令创建的仓库，包含了完成的项目历史和元数据，是Git储存代码和版本信息的主要位置

回退版本

git reset

  • git reset --soft : 保存工作区和暂存区的所有修改内容
  • git reset --hard：丢弃工作区和暂存区的所有内容
  • git reset --mixed：只保留工作区里面的内容（默认参数）

查看差异

git diff 它可以查看文件在工作区、暂存区以及版本库之间的差异，还可以查看文件在版本之间、不同分支之间的差异。

  • git diff 默认比较工作区和暂存区之间的内容
  • git diff --cache 比较暂存区和版本库之间的差异

删除文件

  • 在本地工作区中删除文件

  • 使用git add .来更新工作区\

  • git rm 从工作区和暂存区同时删除，要记得提交

忽略文件

  • 系统或者软件自动生成的文件
  • 编译中间产生的文件文件和结果文件
  • 运行时生成的日志文件、缓存文件、临时文件
  • 涉及身份、密码、口令、密钥等敏感信息文件

GIT Stash

git stash

保存当前工作进度,会把暂存区和工作区的改动保存起来.执行完这个命令后，在运行git status命令，就会发现当前是一个干净的工作区，没有任何改动。
当然，我们一般都是使用 git stash save ‘备注’  可以添加一些注释，方便查阅。

git stash list

显示保存进度的列表。也就意味着，git stash命令可以多次执行。

git stash pop

git stash pop 恢复最新的进度到工作区。git默认会把工作区和暂存区的改动都恢复到工作区。
git stash pop --index 恢复最新的进度到工作区和暂存区。（尝试将原来暂存区的改动还恢复到暂存区）
git stash pop stash@{1}恢复指定的进度到工作区。stash_id是通过git stash list命令得到的
通过git stash pop命令恢复进度后，会删除当前进度。

git stash apply

git stash apply stash@{stash_id}
除了不删除恢复的进度之外，其余和git stash pop 命令一样。

git stash drop 

删除一个存储的进度。如果不指定stash_id，则默认删除最新的存储进度。

git stash clear

删除所有存储的进度。

git stash show

显示与当前分支差异

如：git stash show stash@{stash_id} 加上 -p 可以看详细差异
