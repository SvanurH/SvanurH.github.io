MSF初识

MSF的文件结构

可执行文件

  • msfvenom 可以帮我们生成shell
  • msfupdate 更新
  • msfconsole 交互命令行
  • msfdb 连接数据库，当队友队员资产多的时候可以加入同一个数据库中

目录结构

  • modules 模块
    • auxiliary 主要包含渗透测试中的一些辅助型脚本
    • encoders 编码工具，用于躲过入侵检测和过滤系统
    • exploit 主要包括exp等等各种漏洞利用的脚本
    • nops 空指令，用于避免IDS/IPS的检测
    • payloads 攻击在和，用于在目标系统上执行的代码
    • post 后渗透模块，成功利用漏洞后执行的进一步操作
  • data 存放Meterpreter Passivex VNC DLLS等工具和数据库文件
  • plugins 插件目录，需要使用load命令加载
  • scripts 存放各种脚本文件
  • tools 存放实用的外部工具

payload中普通shell和meterpreter之间的差异

  • 普通shell
    • 没有集成一些插件，只能运行系统本身的命令
    • 操作指令的数据流是明文，可以看到操作的指令
  • meterpreter
    • 集成了很多的插件
    • 指令经过加密

额外知识

正向连接

正向连接（Forward Connection）：正向连接是一种常见的网络通信模式，其中客户端主动发起连接到服务器或目标系统。正向连接通常用于客户端-服务器通信，客户端主动请求服务或资源，例如网页浏览、电子邮件发送和接收等。在正向连接中，客户端充当主动方，向服务器发起连接请求，然后服务器接受并处理请求。

正向shell

攻击机主动去连接受害主机

正向连接是受害主机监听一个端口，由控制主机主动去链接受害主机的过程，适用于受害者主机具有公网ip的情况下。

nc正向shell

反向连接

受害主机出网，主动连接攻击机

反向连接是控制主机监听一个端口，由受害主机反向连接控制主机的过程。适用于受害主机出网且没有公网ip的情况。例如位于内网可以访问互联网的主机。由于反向连接是被攻击者主动连接攻击者，所有较少受到防火墙限制
