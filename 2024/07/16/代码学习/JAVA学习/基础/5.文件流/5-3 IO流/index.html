5-3 IO流

#JAVA学习

IO流概述

  • I指的是Input,成为输入流:负责把数据读到内存中区
  • O指的是Output,称为输出流:负责写数据出去

IO流的分类

按流的方向分为

stateDiagram-v2
    IO流-->输入流
    IO流-->输出流

按流中数据的最小单位,分为:

stateDiagram-v2
	IO流-->字节流
	IO流-->字符流
	字节流-->适合操作所有类型的文件
	字符流-->只适合操作纯文本文件

IO流总体来看就有四大流

字节输入流
  • 以内存问基准,来自磁盘文件/网络中的数据==以字节的形式读入到内存==中去的流
字节输出流
  • 以内存为基准,把内存中的数据==以字节写出到磁盘文件或者网络中去的流==
字符输入流
  • 以内存为基准,来自磁盘文件/网络中的数据==以字符的形式读入到内存==中取得流
字符输出流
  • 以内存为基准,吧内存中的数据==以字符写出到磁盘文件或者网络介质中==去的流

IO流的体系

stateDiagram-v2
字节流-->字节输入流
字节流-->字节输出流
字节输入流-->InputStream
InputStream-->FileInputStream
字节输出流-->OutputStream
OutputStream-->FileOutputStream
字符流-->字符输入流
字符流-->字符输出流
字符输入流-->Reader
Reader-->FileReader
字符输出流-->Writer
Writer-->FileWriter

IO流-字节流

代码优化

问题:
  • 读取数据性能很差
  • 读取汉字输出会出现乱码,并且无法避免
  • 流使用完成后必须关闭,释放系统资源
每次读取多个字节
一次读取完全部字节
  • 方式一:自己定义一个字节数组与被读取的文件大小一样大,然后使用该字节数组,一次读完文件的全部字节.
  • 方式二:Java官方为InputStream提供了如下方法,可以直接把文件的全部字节读取到一个字节数组中返回
方法名称 说明
public byte[] readAllBytes() throws IOException 直接将当前字节输入流对应的文件对象的字节数据装到一个字节数组中返回
  • 如果文件过大,创建的字节数组也会过大,可能引起内存溢出

读写文本内容更适合用字符流,数据转移适合用字节流

案例:文件复制

字节流非常适合做一切文件复制的操作

任何文件的底层都是字节,字节流做复制,是一字不漏的转移完全部字节,只要复制后的文件格式一致就没问题

释放资源的方式

[[#案例 文件复制]]的代码可能出现问题,如果close()之前的代码出现异常,那么关闭流的代码就无法执行

try-catch-finally

  • finally 代码区的特点:无论try中的程序是正常执行了,还是出现了异常,最后都一定会执行finally区,除非==JVM终止==
  • 千万不能在finally中返回数据
  • 作用:==一般用于在程序执行完成后进行资源的释放操作==

try-with-resource

finally 代码比较臃肿

JDK 7开始提供了更简单的资源释放方案

该资源使用完毕后,会自动调用其close()方法,完成对资源的释放

  • try小括号中只能放置资源对象
    • 资源对象:占用系统资源,资源都会实现AutoCloseable接口

IO流-转换流

不同编码读取会出现乱码的问题

  • 如果代码编码和被读取的文本文件的编码是一致的,使用字符流读取文本文件时不会出现乱码
  • 如果代码编码和被读取的文本文件的编码是不一致的,使用字符流读取文本文件时就会出现乱码

字符输入转换流

InputStreamReader(字符输入转换流)
  • 解决不同编码时,字符流读取文本内容乱码的问题
  • 解决思路:先获取文件的原始字节流,再将其按照真实的字符集编码转换成字符输入流,这样字符输入流中的字符就不乱码了
构造器 说明
public InputStreamReader(InputStream is) 把原始的字节输入流,按照代码默认编码转成字符输入流(与直接使用FileReader的效果一样)
public InputStreamReader(InputStream is,String charset) 把原始的字节输入流,按照==指定字符集编码==成字符输入流

字符输出转换流(了解)

使用String提供的getBytes方法解决

PrintStream和PrintWriter的区别

  • 打印数据的功能上是一模一样的:都是使用方便,性能高效
  • PrintStream继承自字节输出流OutputStream,因此支持写字节数据的方法.
  • PrintWriter继承自字符输出流Writer,因此支持写字符数据出去

补充知识-IO框架

什么是框架

  • 解决某类问题,编写的一套类,接口等,可以理解成一个半成品,大多框架都是第三方研发的.
  • 在框架的基础上开发,可以得到优秀的软件架构,并能提高开发效率
  • 框架的形式:一般是把类,接口等编译成class形式,再压缩成一个.jar结尾的文件发行出去

什么是IO框架

  • 封装了Java提供的对文件,数据进行操作的代码,对外提供了更简单的方式来对文件进行操作,对数据进行读写等.

Commons-io

  • Commons-io是apache开源基金组织提供的一组有关IO操作的小框架,目的是提高IO流的开发效率
