4-4 集合补充知识

#JAVA学习

适合场景

  1. 如果希望记住元素的添加顺序，需要存储重复的元素，又要频繁的根据索引查询数据？
    1. 用ArrayList集合（有序、可重复、有索引），底层基于数组
  2. 如果希望记住元素的添加顺序，且增删首位数据的情况比较多？
    1. 用LinkedList集合（有序、可重复、有索引），底层基于双链表实现的。
  3. 如果不在意元素顺序，也没有重复元素需要存储，只希望增删改查都快？
    1. 用HashSet集合（无序、不重复，无索引），底层基于哈希表实现的
  4. 如果希望记住元素的添加顺序，也没有重复元素需要存储，且希望增删改查都快？
    1. 用LinkedHashSet集合（有序、不重复、无索引），底层基于哈希表和双链表
  5. 如果要对元素进行排序，也没有重复元素需要存储？且希望增删改查都快？
    1. 用TreeSet集合，基于红黑树实现

注意事项

集合的并发修改异常问题

  • 使用迭代器遍历集合时，又同时在删除集合中的数据，程序就会出现并发修改异常的错误。
  • 由于增强for循环遍历集合就是迭代器遍历集合的简化写法,因此,使用增强for循环遍历集合,又同时删除集合中的数据时,程序也会出现并发修改异常错误
如何解决

增强for循环和Lambda没办法处理并发修改异常问题

怎么保证遍历集合同时删除数据不出现bug

  • 使用迭代器遍历集合,只用迭代器自己的删除方法删除数据即可
  • 使用for循环遍历时:可以倒着遍历并删除,或者从前往后遍历,每次删除元素后做i–操作

Collection的其他相关知识

前置知识:可变参数

  • 就是一种特殊的形参,定义在方法、构造器的形参列表里，格式是：数据类型…参数名称;
可变参数的特点和好处
  • 特点：可以不传数据给它，可以传一个或者同时传多个数据给它；也可以传一个数组给它。
  • 好处：常常用来灵活的接收数据
注意事项
  • 一个形参列表中，只能有一个可变参数
  • 可变参数必须方法形参列表的最后面

Collections

  • 是一个用来操作集合的工具类

Collections提供的常用静态方法

方法名称 说明
public static <T> boolean addAll(Collection<? super ?> c,T...elements 给集合批量添加元素
public static void shuffle(List<?> list) 打乱List集合中的元素顺序
public static <T> void sort(List<T> list) 对List集合中的元素进行升序排序
public static <T> void sort(List<T> list,Comparetor<? super T> c) 对List集合中的元素,按照比较器对象指定的规则进行排序
