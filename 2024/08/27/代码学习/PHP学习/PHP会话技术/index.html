$_COOKIE

setcookie(name,value,expire,path,domain,secure); 设置cookie值
name 必需。规定 cookie 的名称。
value 必需。规定 cookie 的值。
expire 可选。规定 cookie 的过期时间。

time()+36002430 将设置 cookie 的过期时间为 30 天。如果这个参数没有设置，那么 cookie 将在 session 结束后（即浏览器关闭时）自动失效。
path 可选。规定 cookie 的服务器路径。

如果路径设置为 “/“，那么 cookie 将在整个域名内有效.如果路径设置为 “/test/“，那么 cookie 将在 test 目录下及其所有子目录下有效。默认的路径值是 cookie 所处的当前目录。
domain 可选。规定 cookie 的域名。

为了让 cookie 在 example.com 的所有子域名中有效，您需要把 cookie 的域名设置为 “.example.com”。当您把 cookie 的域名设置为 www.example.com 时，cookie 仅在 www 子域名中有效。
secure 可选。规定是否需要在安全的 HTTPS 连接来传输 cookie。如果 cookie 需要在安全的 HTTPS 连接下传输，则设置为 TRUE。默认是 FALSE。

setcookie("aaa","",time()-60);删除cookie，设置aaa过期时间为过去
第一次设置cookie，在当前页面是无法取出的

$_SESSION

cookie保存在浏览器端，session是保存在server端，session必须依靠cookie才能实现

使用session时，必须使用session_start();

session 依赖于cookie

session依赖于cookie的原因主要是为了在多个HTTP请求之间保持状态。HTTP是无状态协议，这意味着每个请求都是独立的，服务器不会自动记住之前的请求信息。为了实现用户会话管理，PHP引入了session机制。具体来说，PHP的session依赖于cookie来传递session ID，这是保持用户会话状态的关键。

具体原因如下：

  1. 传递Session ID： 每个会话在服务器端都有一个唯一的标识符，即session ID。PHP使用cookie（默认情况下，名为PHPSESSID）在客户端和服务器之间传递这个session ID。客户端在第一次请求时，服务器会生成一个session ID，并通过HTTP响应头中的Set-Cookie字段将其发送给客户端。客户端浏览器接收到这个cookie后，会在后续的每个请求中自动包含这个cookie，从而将session ID发送回服务器。

  2. 识别用户会话： 服务器通过接收到的session ID，能够识别不同的用户会话，并从服务器端的存储（如文件系统或数据库）中检索相应的会话数据。这样，即使用户发起了多个请求，服务器也能够将这些请求关联到同一个会话中，从而实现状态的保持。

  3. 安全性和隐私： 通过使用cookie传递session ID，避免了在URL中传递session ID，减少了session ID暴露在URL中的风险。此外，cookie可以设置为HttpOnly和Secure属性，从而增强session的安全性和隐私保护。

PHP中session依赖于cookie的主要原因是为了在多个HTTP请求之间保持用户会话状态。通过cookie传递session ID，服务器能够识别不同的用户会话，并从服务器端存储中检索相应的会话数据，从而实现状态保持和用户会话管理。
